Documentation

How the C-ADAS workflow behaves from upload to alert.

This page explains the main objects in the system and the easiest browser-only demo path for presentations or internal testing.

Core Objects

Keep the language simple

Vehicle Code: the car identity used for hazard verification.
Upload Code: one background video processing job.
Hazard Code: one grouped road issue built from one or more reports.
Geofence: a protective zone, manual or auto-generated from a verified hazard.
Browser Demo

Recommended test flow

  1. Register two users and assign different vehicle codes.
  2. Upload the same video from both users.
  3. Open Hazards and confirm the same issue becomes verified.
  4. Let the backend auto-create the hazard geofence.
  5. Use Check Location to trigger hazard and geofence alerts in the browser.

ML

Inference Notes

The upload worker uses the trained YOLOv8 pothole weights first and falls back only if the project-specific weight file is not present.

Data

Validation Rules

A hazard is verified when at least two distinct vehicle codes report the same area within the configured radius and time window.
